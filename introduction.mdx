Ship documentation your users will actually read. Mintlify connects directly to your Git repository so every `git push` triggers a live update — no separate CMS, no manual deploys, and no wrestling with static-site tooling. You get a fully branded, searchable docs site with a rich component library, built-in AI tools, and an instant local preview, all driven by MDX files you already control.

## How it works

<Steps>
  <Step title="Connect your repository">
    Link your GitHub (or GitLab) repository to Mintlify. Mintlify watches your default branch and triggers a new build on every push.
  </Step>
  <Step title="Write MDX pages">
    Add `.mdx` files to your repo. Each file becomes a page in your docs site. Use front matter to set the title, description, and sidebar label for every page.
  </Step>
  <Step title="Configure docs.json">
    Define your site's navigation structure, branding (colors, logo, fonts), and integrations inside a single `docs.json` file at the root of your repo.
  </Step>
  <Step title="Push to deploy">
    Commit your changes and push. Mintlify builds and publishes your updated docs automatically — no CI pipelines or extra deployment steps required.
  </Step>
</Steps>
