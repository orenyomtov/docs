In this guide, you'll connect a GitHub repository to Mintlify, preview your docs locally using the CLI, and deploy your first live docs site — all in under five minutes.

<Steps>
  <Step title="Create your docs repo">
    Go to [https://dashboard.mintlify.com](https://dashboard.mintlify.com) and click **New project**. Connect your GitHub account and either use the Mintlify starter template or point Mintlify at an existing repo.

    The starter template includes everything you need to hit the ground running:
    - A pre-configured `docs.json`
    - An `index.mdx` home page
    - Example content pages to build from
  </Step>

  <Step title="Install the CLI">
    Install the Mintlify CLI globally so you can preview your docs on your machine before pushing changes.

    ```bash
    npm i -g mint
    ```
  </Step>

  <Step title="Preview locally">
    Navigate to your repo root — the directory that contains your `docs.json` file — and start the local preview server.

    ```bash
    mint dev
    ```

    Your docs site will be available at [http://localhost:3000](http://localhost:3000). The server hot-reloads as you edit files, so you can see changes instantly without restarting.
  </Step>

  <Step title="Deploy your changes">
    Push your changes to the branch connected to your Mintlify project. Mintlify detects the push, builds your site, and deploys it automatically — no manual steps required.

    Once the build completes, view your live site URL from the [Mintlify dashboard](https://dashboard.mintlify.com).
  </Step>
</Steps>

<Tip>
  Run `mint update` if your local preview isn't working to get the latest CLI version.
</Tip>
